# 🚀 Remnawave Node Multi-Protocol Setup

Подготовка ноды Remnawave для работы с дополнительными транспортами:

* 🚀 VLESS Reality (базовый)
* ⚡ Hysteria2 (QUIC)
* 🔄 VLESS gRPC + Reality
* 🌐 VLESS XHTTP + Reality

---

# ⚠️ Требования

Перед запуском скрипта нода должна быть:

✅ Добавлена в панель Remnawave

✅ Установлена через reverse-proxy от eGame

Документация:

https://github.com/eGamesAPI/remnawave-reverse-proxy/blob/main/README-RU.md

# 📦 Что делает скрипт

### Проверка окружения

* Проверяет запуск от root
* Проверяет наличие Docker
* Проверяет доступность Docker daemon
* Проверяет наличие контейнера `remnanode`
* Проверяет что контейнер запущен
* Проверяет доступность `/dev/shm`
* Проверяет наличие сертификатов Let's Encrypt
* Проверяет работу VLESS Reality на порту `443/tcp`

### Настройка протоколов

Скрипт позволяет выбрать:

#### Hysteria2

* Открывает `443/udp`
* Копирует сертификаты в `/dev/shm`
* Настраивает автоматическую синхронизацию сертификатов
* Создаёт Cron-задачи для обновления сертификатов
* При необходимости перезапускает контейнер

#### VLESS gRPC + Reality

* Открывает `8443/tcp`

#### VLESS XHTTP + Reality

* Открывает `4443/tcp`

### Работа с сертификатами

Скрипт автоматически:

* Копирует сертификаты Let's Encrypt в `/dev/shm`
* Проверяет актуальность сертификатов
* Создаёт задачу обновления после перезагрузки сервера
* Создаёт ежедневную задачу обновления сертификатов
* Перезапускает контейнер после обновления сертификатов

### Firewall

Если установлен и активен UFW:

* Открывает необходимые порты автоматически
* Не создаёт дубликаты правил

# 🔧 Порядок настройки

### 1. Подготовить сервер

Запустить скрипт установки.

### 2. Выбрать необходимые протоколы

Доступные варианты:

* Hysteria2
* VLESS gRPC + Reality
* VLESS XHTTP + Reality
* Все протоколы одновременно

### 3. Открыть конвертер конфигов

https://rezzosoft.ru/converter.html

### 4. Скопировать конфиг ноды из Remnawave

Открыть профиль ноды и скопировать текущий JSON-конфиг.

### 5. Сгенерировать мульти-протокольный конфиг

Вставить JSON в конвертер и выбрать нужные транспорты.

### 6. Создать новый Config в панели

Вставить результат конвертации в новый Config.

### 7. Привязать Config к ноде

Выбрать новый Config в настройках ноды.

### 8. Создать необходимые Host

Для каждого используемого транспорта.

### 9. Добавить inbound'ы в Squad

Добавить нужные inbound'ы в соответствующий Squad.

### 10. Готово 🎉

Нода сможет одновременно обслуживать:

* VLESS Reality
* Hysteria2
* VLESS gRPC + Reality
* VLESS XHTTP + Reality

# 🔄 Идемпотентность

Скрипт безопасно запускать повторно.

Он:

* Не создаёт дубликаты Cron-задач
* Не открывает уже открытые порты
* Не копирует сертификаты без необходимости
* Не выполняет лишние перезапуски контейнера
